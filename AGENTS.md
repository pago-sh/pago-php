# pago-php — Agent Guide

`pago-sh/sdk` (main class `Pago`) — the official pago.sh SDK for PHP. **Generated by Speakeasy** from the pago.sh OpenAPI spec.

## ⚠️ Generated code
Most files are generated and **overwritten on regeneration** — do not hand-edit them. Source of truth: the OpenAPI spec + `.speakeasy/gen.yaml` (SDK `version`); the version constant lives in `src/SDKConfiguration.php` (`$sdkVersion`). Regeneration is manual (CI workflows removed).

## Quick commands
```bash
composer install
vendor/bin/phpstan analyse    # static analysis (if configured)
```
Published via **Packagist**: register the repo on packagist.org, then release with `git tag v1.0.0 && git push origin v1.0.0` (Composer resolves versions from git tags).

## Structure
- `src/` — generated SDK (`Pago.php`, `Models/`, `SDKConfiguration.php`) · `.speakeasy/` — gen.yaml + overlays

## Git Workflow

Branch model — the same across every repo in the pago.sh ecosystem:

- **`main`** — production. Releases (git tags) run from `main`. Never commit or push directly to `main`.
- **`develop`** — integration & testing. Validated here before reaching `main`.
- **`feat/<name>`** / **`fix/<name>`** — branch off `develop`, merge back into `develop`.

**Flow:** `feat/*` or `fix/*` → **`develop`** (test) → merge into **`main`** → production.
